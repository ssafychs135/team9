1. 도입 (Introduction)
  "안녕하세요, AI 기술을 활용한 유튜브 리뷰 요약 및 공유 플랫폼, 'ReviewSite'의 발표를
  맡은 팀9 발표자 [본인이름]입니다. 저희는 '정보의 홍수 속에서 어떻게 하면 효율적으로
  가치 있는 리뷰를 소비하고 공유할 수 있을까?'라는 고민에서 이 프로젝트를 시작했습니다."

  2. 프로젝트 개요 (Overview)
  "ReviewSite는 단순히 텍스트 리뷰를 올리는 공간이 아닙니다. 유튜브 링크 하나만으로 AI가
  영상의 핵심 내용을 요약해 주며, 전문적인 iPhone 지식을 갖춘 AI 챗봇과 실시간으로
  상담할 수 있는 스마트한 리뷰 커뮤니티입니다."

  3. 주요 기능 시연 (Key Features) - 화면 시연과 병행
  "저희 서비스의 주요 기능을 소개해 드리겠습니다.
   * 첫 번째는 AI 자동 요약 리뷰입니다. 사용자가 유튜브 URL을 입력하고 '요약 요청'을
     누르면, 백엔드 워커가 영상의 자막을 추출하고 LLM을 통해 마크다운 형식의 깔끔한
     요약본을 생성합니다.
   * 두 번째는 지능형 챗봇, 'AI Genius'입니다. 단순한 대화형 AI를 넘어, 저희가 직접
     크롤링한 아이폰 스펙 데이터를 기반으로 정확한 정보를 제공하는 RAG 기반 챗봇입니다.
   * 세 번째는 사용자 편의성입니다. Apple의 디자인 철학을 계승한 미려한 UI와 다크 모드,
     그리고 이번에 새롭게 구현한 커스텀 모달 시스템을 통해 브라우저 기본 UI를 벗어난
     일관된 사용자 경험을 제공합니다."

  4. 시스템 아키텍처 (Architecture) - 아키텍처 도식화 슬라이드
  "기술적인 구조를 설명드리겠습니다. 저희는 확장성과 효율성을 위해 마이크로서비스 지향
  아키텍처를 채택했습니다.
   * Frontend는 Vue 3와 Pinia를 사용하여 상태 관리를 최적화했고, 컴포넌트 기반 설계를
     통해 코드 재사용성을 높였습니다.
   * Main Backend는 Django REST Framework를 사용했습니다. 특히 인증 라이브러리를 활용한
     보안 강화와, 중첩 라우팅을 통한 계층적 API 설계에 집중했습니다.
   * Worker Backend는 Node.js로 분리했습니다. 무거운 자막 추출 및 요약 로직을 분리하여
     메인 서버의 부하를 줄이고 독립적인 확장이 가능하도록 했습니다."

  5. 핵심 기술: RAG와 챗봇 (Core Tech)
  "저희 프로젝트의 기술적 자부심은 RAG(검색 증강 생성) 패러다임의 적용입니다.
  일반적인 LLM은 최신 기기 정보에 대해 할루시네이션(환각)을 일으킬 수 있습니다. 이를
  방지하기 위해 저희는 ChromaDB라는 벡터 데이터베이스에 최신 아이폰 스펙 데이터를
  저장했습니다. 사용자가 질문하면 LangChain이 데이터베이스에서 관련 정보를 먼저
  검색하고, 그 근거를 바탕으로 답변을 생성하기 때문에 매우 높은 신뢰도의 정보를 제공할
  수 있습니다."

  6. 개발 과정의 도전과 해결 (Refactoring)
  "개발 과정에서 UI/UX의 디테일을 높이기 위해 많은 노력을 기울였습니다. 
  기존의 파편화된 CSS 코드를 전역 변수와 유틸리티 클래스로 통합하는 대대적인 리팩토링을
  거쳤으며, window.alert 같은 투박한 기본 기능을 Promise 기반의 커스텀 모달로 직접
  구현하여 사용자 흐름이 끊기지 않도록 개선했습니다. 또한, 데이터 덤프 시 로그가 섞이는
  문제를 해결하기 위해 표준 입출력 채널을 제어하는 등 안정적인 개발 환경 구축에도
  힘썼습니다."

  7. 결론 및 향후 계획 (Conclusion)
  "ReviewSite는 기술이 사용자에게 줄 수 있는 '시간의 가치'를 증명하고자 했습니다.
  앞으로는 더 다양한 카테고리의 영상 요약을 지원하고, 사용자 맞춤형 추천 기능을 고도화할
  계획입니다. 이상으로 팀9의 발표를 마치겠습니다. 감사합니다!"
